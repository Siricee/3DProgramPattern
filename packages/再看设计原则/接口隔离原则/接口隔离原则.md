# 1.4 接口隔离原则

## 定义

模块间的依赖关系应该建立在最小的接口上


## 补充说明

应该将庞大的接口拆分成更小的接口

“接口”属于面向对象编程的概念，它在函数式编程中的对应是“类型”。两者的对比代码如下：
接口
```ts
export interface Interface1 {
    value1: number
    func1: () => string
}
```
类型
```ts
export type type1 = {
    value1: number
    func1: () => string
}
```

因为这两者其实是等价的，所以说接口隔离原则应用在函数式编程中就是指类型应该保持最小

值得注意的是：
类型名首写字母是小写，而接口名首写字母是大写
